MY INITIAL POST

    AUTHENTICATION AND AUTHORIZATION; AN ESSENTIAL FOR NETWORK AND SOFTWARE SECURITY.

    Security is a crucial part of any application or environment (Bullock and Benford, 1999) and granting access to these secured applications and the environment needs to be controlled; who gets access to what? What are the rules? How is it tracked? These are managed through an access control system providing access to restricted areas of the business (SailPoint, 2021).” Access control is governed according to the space within which subjects and objects reside, and the ability to traverse space to get close to an object” by Bullock and Benford, 1999. Who(subject) gets access to resources(object) is validated through authentication and authorisation processes.

    Authentication, verifying who someone is, has long passed username and passwords to more advanced Artificial intelligence and deep learning technologies. The application of multi-factor authentication verifying an individual identity based on two to three different parameters, something you (have, know and are) works based on artificial intelligence and deep learning, where data, logs, transactions, and real-time communications are analysed to detect an intruder (Okta, 2018).

    Hardware embedded chips are evolving technologies that authenticate users using multiple levels and methods. An example is the sixth generation vPro Chips by intel (IFF Lab, 2019).

    Authorisation is to which asset one has permission to access through settings supported by security teams (Okta, 2018). The more often used to perform these authorisation processes include Role-Based Access Control, Discretionary based access control, Attribute-based access control, Rule-Based Access Control. These strategies help application developers control the access to the resources depending on the applied method and to what level the user is permitted.

    Authentication and authorisation are vital in access control management, verifying a user’s identity effectively, thus allowing authenticated users to access protected resources depending on the given authority level; these ensure that security is maintained both in the network and the software to be accessed (Wang, Bo and Zhang, 2013).

Hi, I agree with your post. Firstly, the importance of authentication and authorisation is well documented. Summary Post

The importance of control mechanisms (how, who, and to what extent) access is granted into a network cannot be overemphasised, authenticating a user into the network, as discussed during this period. One of the weakest links in security is the user; as pointed out by Richard (Tolofari and Meadows, 2021), users need to be educated on how to create their passwords, not using conventional passwords like 12345. The Open Web Application Security Project (OWASP), among other bodies, provides guidance on minimum criteria for a strong password, that passwords cannot be shorter than eight and should be enforced by applications (NIST SP800-63B). Maximum password length should not be set too low and more; ensure credentials change when there is compromise or password leak. A password alone cannot guarantee the security of a network; the combination of other factors- multifactor authentication mechanisms should be implemented." The potential in multifactor authentication security features will not be exhausted anytime soon as long as there are unique and differing attributes in life. The challenge may only lie in the ability to integrate new identification features into authorisation and authentication" by Jubet., Authentication methods are based on only the possession of credentials issued to the user. The verification is performed on credentials offered at present, not considering how or when the user obtained the credentials (Hasan and Khan, 2015). Nitin also pointed out during the discussions the importance of Adaptive Multifactor Authentication. Where contexts like location awareness, device fingerprinting or impossible velocity are added to the multifactor authentication (Stjernlof and Vries, 2020), these adaptive multifactor authentications using artificial intelligence is a step further to harden the authentication system. Isioma, during the discussions, highlighted that authenticating a user before authorisation; granting access to data or resources in the network is crucial, not the other way round. To what criteria and rules are the users assessed (Tolofari and Okafor -Tolofari, 2020), like Discretionary access control: where the owner of the system determines who gets access to the system; Role-based access control, where access is granted based on the user role, Mandatory access control, which allows access based on clearance level are all among other criteria to authorise a user on what resources they can access in the network. Authentication and authorisation are essential to network security, how they are implemented, and to what extent should depend on the asset's value. The efficiency depends on the technology used to enforce this authentication and authorisation process.

◉COMPROMISING THE CLIENT INVOLVES

Shoulder surfing

This is when an intruder tries using social engineering by spying on the victim to get his credentials through visual hacking (McGinnis, 2017).

User unattended logging on

sessions timeout should be implemented to mitigate this threat. In addition, screen shields should be placed between patients and receptionists to avoid shoulder surfing.

Obtain a valid username/password

obtaining a username and password through social engineering or reconnaissance is a threat to the web appointment scheduling application. Credentials should be encrypted when stored in use or transit, and strong authentication measures like two factor or multi-factor authentication should be implemented to authenticate users to the system.

Social engineering

This is one of the methods threat actors use to access the network, like phishing attempts on accounts, tailgating, and shoulder surfing. Therefore, users should be educated on social engineering activities and report any suspicious activities to the administrators or authorities.

A network interception attack

occurs when a threat actor eavesdrops on the network, intercepting and gathering information so that he can gain access. This is prevented by encrypting data stored or used in transit.

Keylogging

this is a programme that records keystrokes from keyboards and other inputs such as mouse movement and touchscreen inputs for the attacker to analyse the user's information (Chapple and Seidl, 2021).

Phishing emails

using emails to steal user identity online. A victim receives an email that seems to come from a bank or a financial institution. The email links to a website where the user is asked to provide account details. These credentials are then harvested by the threat actor and used for malicious purposes like compromising the web appointment scheduling application (Lininger and Vines, 2005).

Infect with malware

Infecting the system with malicious code that disrupts the proper functioning of the scheduling system or tries to steal sensitive or personal information from the system.

Deliver malware through email attachments

most malware is delivered through email attachments like phishing campaigns in the form of embedded URLs. When clicked on by the user, this malware is then downloaded to the network. The malware then sits on the network and exploits or exfiltrates information to the threat actor.

Gain local access; Obtain administrator username/password

when a user is compromised, the threat actors will then move laterally to gaining privilege access to the network to obtain administrators credentials and then gaining access to the server.

Steal client Devicessmartphones

since most times users can use their mobile devices to access the application, there is the vulnerability that their device can be stolen, and an attacker gets hold of their credentials to infiltrate the network.

These are attacks that a threat actor perpetuates against a server to infiltrate the server and compromise the network. These attacks are

An attacker who does not have access to the machine being targeted uses remote exploitation techniques to exploit a product or a component of a product. Antivirus software can be targeted remotely also.

Default accounts and passwords are widely used and risky. flooding the application with too many requests at the same time will cause a denial of service to legitimate users on the network.

PREVENTION AND MITIGATIVE MEASURES AGAINST THESE ATTACKS

The following is a list of potential mitigation and preventive measures for dealing with the identified attacks.

  • Authentication: To use a client, users must be authenticated. As a standard practice, a strong password policy and multifactor authentication should grant access to network users.

  • User Authorisation: Access policies based on roles should be created.

  • Principles such as least privileges and separation of duties must be implemented.

  • Users who have been granted access, roles with the bare minimum of privileges
    • should be assigned to prevent privilege escalation if their accounts are compromised.

  • Auditing: Every action must be documented and accounted for.

  • Malware Detection: Different varieties of malware can affect clients. As a result,
    • malware detection programmes should be installed and updated regularly.

  • Logout Policy: A client should be logged out after a period of inactivity.

  • Access Revocation Policy: It is quite easy to steal portable clients. As a result, there should be a way to disable their access to the system.

  • Security for servers should involve implementing defence-in-depth measures, such as firewalls, intrusion prevention systems, and intrusion detection systems.

  • Physical security prevents intruders from removing databases by protecting the servers.

  • Encryption: Network assaults can be significantly mitigated by encrypting all communications in transit, in use, and in storage using appropriate cryptographic protocols like TLS/SSL.

  • Security Awareness (Education): All authorised users (Administrators) must complete a security awareness programme. They should learn about attacks, consequences, and best practices in this system.
